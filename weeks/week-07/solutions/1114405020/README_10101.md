# UVA 10101 - 木棒遊戲

## 題目概述

### 問題描述

這是一個古老的遊戲：用**木棒**在桌上拼出一個**不成立的等式**，通過移動**恰好一根木棒**使得等式成立。

**核心限制：**
- 式子只包含加號 `+` 和減號 `-`（包括負號），以及一個等號 `=`
- 只能移動用來構成**數字的木棒**，不能移動運算符
- 所有數字必須嚴格符合**七段顯示器**的標準表示
- 修改前的等式中的數不能以 `0` 開頭（除了 `0` 本身），但修改後的數可以以 `0` 開頭

### 輸入規格
- 一行字符串，包含等式，以 `#` 結尾
- 格式例如：`1+1=3#`
- 字符串長度 ≤ 1000

### 輸出規格
- 若有解，輸出修正後的等式（格式同輸入，以 `#` 結尾）
- 若無解，輸出 `No`（大寫 N，小寫 o）

### 範例

```
輸入：  1+1=3#
輸出：  1+2=3#    （把一個 1 改為 2）

輸入：  2+2=5#
輸出：  No        （無法通過移動一根木棒使等式成立）
```

## 演算法分析

### 七段顯示器映射

每個數字由七段木棒組成，標記如下：

```
     a
    ---
   |   |
  f|   |b
    -g-
   |   |
  e|   |d
    ---
     c
```

**各數字的木棒組成：**

| 數字 | 木棒 | 個數 |
|------|------|------|
| 0 | {a,b,c,d,e,f} | 6 |
| 1 | {b,c} | 2 |
| 2 | {a,b,d,e,g} | 5 |
| 3 | {a,b,c,d,g} | 5 |
| 4 | {b,c,f,g} | 4 |
| 5 | {a,c,d,f,g} | 5 |
| 6 | {a,c,d,e,f,g} | 6 |
| 7 | {a,b,c} | 3 |
| 8 | {a,b,c,d,e,f,g} | 7 |
| 9 | {a,b,c,d,f,g} | 6 |

**重要觀察：**
- 數字 1 最少（2 根），數字 8 最多（7 根）
- 相同木棒數的不同數字可以互相轉換（移動 0 根木棒）
- 相差 1 根木棒的數字可以通過移動 1 根木棒轉換

### 核心演算法

採用**暴力列舉 + 驗證**的策略：

1. **提取數字**
   - 解析等式，提取所有數字和運算符位置
   - 記錄每個數字的木棒表示

2. **列舉移動**
   ```
   for 每個數字位置 i:
       for 該位置的每根木棒 segment:
           臨時移除該木棒
           if 移除後仍是有效數字:
               for 其他位置 j:
                   for 每個可能的木棒:
                       嘗試添加該木棒
                       if 添加後仍是有效數字:
                           重建等式並驗證
   ```

3. **驗證等式**
   - 解析左側表達式，計算結果
   - 解析右側表達式，計算結果
   - 比較兩邊是否相等

### 時間複雜度

```
D = 等式中的數字個數（≤ 20）
S = 每個位置最多的木棒數（≤ 7）

複雜度：O(D × S × D × S × D) = O(D^3 × S^2)
     ≈ O(20^3 × 7^2) ≈ 400 萬次操作

可在實際時間內完成
```

### 空間複雜度

```
O(D × S) ≈ O(D) ≈ O(20)
主要用於儲存數字的木棒集合
```

## 實現細節

### 關鍵函數

#### 1. `digit_to_matchstick(digit: str) -> Set[str]`
將單個數字轉換為木棒集合表示。

**時間複雜度：** O(1)

**例子：**
```python
digit_to_matchstick('0') → {'a', 'b', 'c', 'd', 'e', 'f'}
digit_to_matchstick('1') → {'b', 'c'}
digit_to_matchstick('8') → {'a', 'b', 'c', 'd', 'e', 'f', 'g'}
```

#### 2. `matchstick_to_digit(matchsticks: Set[str]) -> Optional[str]`
將木棒集合轉換回數字。

**時間複雜度：** O(1)

**說明：**
- 如果木棒集合對應有效數字，返回該數字
- 否則返回 `None`

**例子：**
```python
matchstick_to_digit({'a', 'b', 'c', 'd', 'e', 'f'}) → '0'
matchstick_to_digit({'b', 'c'}) → '1'
matchstick_to_digit({'x', 'y'}) → None  # 無效
```

#### 3. `parse_expression(expr: str) -> Optional[int]`
計算數學表達式的值。

**時間複雜度：** O(|expr|)

**說明：**
- 從左到右掃描表達式
- 處理加法和減法
- 處理開頭的負號（負數）

**例子：**
```python
parse_expression("12+34") → 46
parse_expression("9-3") → 6
parse_expression("-5+3") → -2
```

#### 4. `extract_numbers(equation: str) -> tuple`
提取等式中的數字，返回 (數字列表, 左側模板, 右側模板)。

**說明：**
- 使用模板表示數字位置，便於後續重建
- 模板中用 `{i}` 表示第 i 個數字的位置

**例子：**
```
輸入：  "12+34=46"
輸出：  (['1','2','3','4','4','6'], 
          '{0}{1}+{2}{3}', 
          '{4}{5}')
```

#### 5. `solve_matchstick(equation: str) -> str`
主求解函數。

**演算法流程：**
1. 清理輸入（移除 `#` 及後續內容）
2. 提取所有數字和模板
3. 列舉所有可能的木棒移動
4. 驗證每個修改是否產生成立的等式
5. 返回第一個成立的等式，或 `"No"`

**時間複雜度：** O(D^3 × S^2)

## 關鍵演算法技巧

### 1. 七段顯示器映射

使用集合表示木棒，便於高效的添加/移除操作：

```python
DIGIT_SEGMENTS = {
    '0': {'a', 'b', 'c', 'd', 'e', 'f'},
    '1': {'b', 'c'},
    ...
}

# 反向映射
SEGMENTS_TO_DIGIT = {frozenset(v): k for k, v in DIGIT_SEGMENTS.items()}
```

**優點：**
- 快速查詢：O(1)
- 支持集合操作（交、並、差）
- 易於檢查有效性

### 2. 模板替換

使用模板避免複雜的字符串操作：

```python
# 提取時標記位置
left_template = "{0}{1}+{2}{3}"  # 12+34 的模板

# 修改時直接替換
new_left = left_template.replace("{0}", '2')  # 改為 22+34
```

**優點：**
- 清晰易懂
- 避免下標錯誤
- 便於驗證

### 3. 回溯搜索

使用回溯在嘗試失敗時恢復狀態：

```python
temp[add_idx].add(add_seg)      # 嘗試添加
# ... 驗證 ...
temp[add_idx].remove(add_seg)   # 回溯
```

**優點：**
- 不需要複製整個狀態
- 內存效率高
- 實現簡潔

## 常見錯誤

### 1. 忽視運算符的不變性
**錯誤：** 試圖修改運算符
**正確：** 只修改數字，運算符保持不變

### 2. 未考慮負號
**錯誤：** 不處理負數情況（如 `-5+3`）
**正確：** 將表達式的負號作為特殊處理

### 3. 無效的木棒組合
**錯誤：** 允許木棒集合對應無效數字
**正確：** 每次修改後檢查對應的數字是否有效

### 4. 忽視 0 開頭的允許性
**錯誤：** 認為 `0` 開頭的數字不合法
**正確：** 修改後的數字可以以 `0` 開頭，但修改前不能

### 5. 移動根數計算錯誤
**錯誤：** 移除和添加不是同一根木棒（實際移動了多根）
**正確：** 從一個位置移除 1 根，在另一位置添加 1 根

## 優化建議
