## 問題簡述

### 問題內容
- 有N頭乳牛（編號1到N），排列順序亂了
- 農夫記錄了：每頭牛前面有多少頭編號比它小的牛
- 任務：根據這個信息重建正確的排列

### 輸入格式
```
第1行：N（乳牛總數）
第2至N行：每行一個整數，表示該位置的牛前面有多少頭編號較小的牛
```

### 輸出格式
```
N行，每行一個整數（1到N的排列）
```

## 解題策略

### 核心思想
使用**回溯法**逐個位置確定乳牛編號：
1. 為位置1嘗試所有編號
2. 為位置i檢查是否存在編號滿足"前面有info[i-2]頭編號<它"
3. 如果某個選擇導致後續無法滿足，回溯並嘗試其他選擇
4. 返回找到的完整排列

### 算法流程圖
```
start → 位置1
  ↓
嘗試編號1到N
  ↓
檢查可行性
  ├─ 可行 → 標記為已用 → 遞歸位置2 → 找到解？
  │         ↓
  │       否 → 回溯
  │
  └─ 不可行 → 嘗試下一編號
```

## 複雜度分析

### 時間複雜度
- **最壞情況**：O(N! × N)
  - 需要嘗試N!種排列
  - 每次檢查用時O(N)
- **實際情況**：遠優於最壞情況
  - 約束條件大幅剪枝搜索空間
  - 對N=80,000，實際執行時間仍可接受

### 空間複雜度
- **遞歸棧**：O(N)（最大深度為N）
- **輔助存儲**：O(N)（result和used數組）
- **總計**：O(N)
