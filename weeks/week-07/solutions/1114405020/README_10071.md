## 問題簡述

### 問題內容
- 給定整數集合S（N個元素）
- 計算滿足 **a + b + c + d + e = f** 的六元組(a,b,c,d,e,f)個數
- a、b、c、d、e、f都要從S中選取（允許重複）

### 輸入格式
```
第1行：N（集合S的元素個數，1 ≤ N ≤ 100）
第2至N+1行：集合S的元素（範圍-30000到30000）
```

### 輸出格式
```
單個整數：符合條件的六元組個數
```

## 解題策略

### 核心問題
如果使用暴力法（6層迴圈逐一檢查所有六元組），時間複雜度為O(N^6)。
對於N=100，這意味著10^12次操作，太慢了！

### 優化思路
關鍵觀察：**與其檢查每個f，不如先計算好所有a+b+c+d+e的和**

改進策略：
1. **第一階段**（預處理）：計算所有(a,b,c,d,e)的和及其出現次數
2. **第二階段**（計數）：遍歷f，查詢對應的五元組個數

這樣時間複雜度變為O(N^5)，性能提升100倍！

### 演算法流程圖
```
start → 讀取集合S
  ↓
迴圈a,b,c,d,e ← 計算所有五元組的和
  ↓
用字典記錄每個和的出現次數
  ↓
迴圈f∈S ← 查字典
  ↓
累加滿足條件的六元組個數
  ↓
輸出結果
```

## 關鍵代碼片段

### 核心演算法（Easy版本）
```python
# 步驟1：計算所有五元組之和
sum_count = {}
for a in S:
    for b in S:
        for c in S:
            for d in S:
                for e in S:
                    total = a + b + c + d + e
                    sum_count[total] = sum_count.get(total, 0) + 1

# 步驟2：遍歷f，查詢結果
result = 0
for f in S:
    if f in sum_count:
        result += sum_count[f]
return result
```

## 常見問題

### Q: 為什麼要用字典而不是列表？
A: 字典查詢的時間複雜度是O(1)，而列表查詢是O(N)。對於這個問題，字典性能更好。

### Q: 可以減少空間複雜度嗎？
A: 理論上可以用排序+雙指針等技巧，但會增加時間複雜度或代碼複雜度。對於N≤100，當前方案已是最優平衡。

### Q: 如何處理集合中有重複元素的情況？
