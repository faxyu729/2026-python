# UVA 10170 - 無限房間旅館

## 題目概述

### 問題描述

HaluaRuti 城有一家奇特的旅館，擁有**無限多間房間**。旅館的住宿規則如下：

**住宿規則：**
- 同一時間只能有一個旅行團住宿
- 每個旅行團早上入住，傍晚退房
- 前一個旅行團退房的隔天早上，下一個旅行團入住
- 每個旅行團的人數比前一個多 1 人（第一個旅行團有 S 人）
- 一個有 n 人的旅行團，會住 n 天

**範例說明：**

若起始旅行團有 4 人，則：
- 第 1~4 天：4 人團隊（4 人 × 4 天）
- 第 5~9 天：5 人團隊（5 人 × 5 天）
- 第 10~15 天：6 人團隊（6 人 × 6 天）
- 第 16~22 天：7 人團隊（7 人 × 7 天）
- ...

### 輸入規格
- 每行包含兩個整數 **S** 和 **D**
  - S：第一個旅行團的人數（1 ≤ S ≤ 10000）
  - D：查詢天數（1 ≤ D < 10¹⁵）
- 輸入直到 EOF 結束

### 輸出規格
- 每行輸出一個整數，表示第 D 天住宿的旅行團人數

### 範例

```
輸入：
4 4
4 5
4 10
1 6

輸出：
4
5
6
3
```

**解釋：**
- 4 4：S=4, D=4 → 第 4 天在 4 人團（第 1-4 天），答案：4
- 4 5：S=4, D=5 → 第 5 天在 5 人團（第 5-9 天），答案：5
- 4 10：S=4, D=10 → 第 10 天在 6 人團（第 10-15 天），答案：6
- 1 6：S=1, D=6 → 第 6 天在 3 人團（第 4-6 天），答案：3

## 演算法分析

### 核心觀察

**旅行團的規律：**
- 第 i 個旅行團有 (S + i - 1) 人
- 第 i 個旅行團住 (S + i - 1) 天

**住宿天數的計算：**
前 i 個旅行團的總住宿天數：

```
sum(i) = S + (S+1) + (S+2) + ... + (S+i-1)
       = i*S + (0+1+2+...+(i-1))
       = i*S + i(i-1)/2
       = i(2S + i - 1)/2
```

### 數學推導

要找第 D 天屬於哪個團隊，需要找到最小的 i 使得：

```
i(2S + i - 1)/2 >= D
i(2S + i - 1) >= 2D
i² + (2S-1)i - 2D >= 0
```

使用二次公式求解：

```
i = (-(2S-1) + sqrt((2S-1)² + 8D)) / 2
```

但這種方法涉及浮點數計算，可能有精度問題。

### 採用的演算法：二分查找

為了避免浮點數精度問題，採用**二分查找**：

1. **初始化搜尋範圍**
   - 左邊界：left = 1
   - 右邊界：right = S + sqrt(2D) + 常數（估計上界）

2. **二分查找**
   ```
   while left < right:
       mid = (left + right) // 2
       if sum(mid) < D:
           left = mid + 1      # 不夠，往右找
       else:
           right = mid         # 夠了，往左找
   ```

3. **返回結果**
   - 最終 left 就是最小的 i 使得 sum(i) >= D
   - 返回第 i 個團隊的人數：S + i - 1

### 時間複雜度分析

```
二分查找：O(log(上界))
                   ≈ O(log(S + sqrt(2D)))
                   ≈ O(log(S + sqrt(10^15)))
                   ≈ O(log(10^7 + 10^8))
                   ≈ O(27) ≈ O(log D)
```

對比暴力法 O(D)，這是巨大的改進。D 最大為 10^15，不可能逐項遍歷。

### 空間複雜度

```
O(1)：僅使用常數個變數
```

## 實現細節

### 核心函數

#### 1. `sum_days(s: int, team_count: int) -> int`

計算前 team_count 個團隊的總住宿天數。

**公式：** `team_count * (2*s + team_count - 1) // 2`

**時間複雜度：** O(1)

**例子：**
```python
sum_days(4, 1) → 4       # 第 1 個團隊：4 人 × 4 天
sum_days(4, 2) → 9       # 前 2 個團隊：4 + 5 = 9 天
sum_days(4, 3) → 15      # 前 3 個團隊：4 + 5 + 6 = 15 天
```

#### 2. `find_group_size(s: int, d: int) -> int`

主求解函數，找到第 D 天的旅行團人數。

**演算法流程：**
1. 使用二分查找找最小的 i 使得 sum_days(s, i) >= d
2. 返回第 i 個團隊的人數：s + i - 1

**時間複雜度：** O(log D)

**例子：**
```python
find_group_size(4, 4)   → 4   # 第 1-4 天是 4 人團隊
find_group_size(4, 5)   → 5   # 第 5-9 天是 5 人團隊
find_group_size(4, 10)  → 6   # 第 10-15 天是 6 人團隊
find_group_size(1, 6)   → 3   # 第 4-6 天是 3 人團隊
```

### 邊界處理

**大數值處理：**
- S 最大：10000
- D 最大：接近 10^15
- 使用 Python 的任意精度整數（自動處理）
- 估計上界時使用 `int(sqrt(2*d))` 確保足夠大

**邊界情況：**
- S=1：最小人數團隊
- S=10000：最大人數團隊
- D=1：第一天（返回 S）
- D=10^15：最大查詢天數

## 關鍵演算法技巧

### 1. 數學公式推導

利用等差數列求和公式，將問題轉化為求解二次不等式：

```
和 = S + (S+1) + ... + (S+i-1)
   = i*S + i(i-1)/2
   = i(2S + i - 1)/2
```

### 2. 二分查找應用

避免浮點數精度問題，同時保持 O(log D) 的效率：

```python
while left < right:
    mid = (left + right) // 2
    if sum_days(s, mid) < d:
        left = mid + 1
    else:
        right = mid
```

### 3. 上界估計

根據二次方程的解估計團隊編號的上界：

```python
right = s + int(math.sqrt(2 * d)) + 100
```

## 常見錯誤

### 1. 浮點數精度問題
**錯誤：** 直接使用二次公式計算 i
```python
i = (-b + sqrt(discriminant)) / (2*a)
```
**問題：** D 很大時，浮點數精度不足

**正確做法：** 使用二分查找

### 2. 邊界判斷錯誤
**錯誤：** 找到 i 後，返回 s + i（應該是 s + i - 1）
**正確：** 團隊編號是 1, 2, 3, ...，第 i 個團隊有 s + (i-1) = s + i - 1 人

### 3. 上界設置不足
**錯誤：** right = d（對超大 D 值不安全）
**正確：** right = s + sqrt(2*d) + 常數

### 4. 整數溢出（某些語言）
**Python：** 使用任意精度整數，不存在此問題

## 優化建議

### 1. 使用二次公式的改進版本

```python
def find_group_quadratic(s, d):
    # 使用二次公式
    a, b, c = 1, 2*s-1, -2*d
    discriminant = b*b - 4*a*c
    i = int((-b + math.sqrt(discriminant)) / (2*a))
    
    # 驗證並調整
    while sum_days(s, i) < d:
        i += 1
    while i > 1 and sum_days(s, i-1) >= d:
        i -= 1
    
    return s + i - 1
```

### 2. 預計算優化

如果有多次查詢，可以預先計算某些值。

### 3. 三分查找

由於 sum_days 是關於 i 的單調遞增函數，也可以使用三分查找（但二分已足夠）。
