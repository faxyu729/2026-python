# UVA 10093 - 炮兵部隊部署問題

## 題目概述

### 問題描述
在 **N×M 的網格地圖**上部署炮兵部隊，最大化部署數量。

**地形類型：**
- `H`（山地）：不能部署炮兵
- `P`（平原）：可以部署炮兵

**攻擊範圍：**
- 炮兵攻擊範圍為：沿橫向左右各2格，沿縱向上下各2格
- 攻擊範圍形成十字形（不包含斜對角）

**約束條件：**
- 任何兩支炮兵都不能互相攻擊
- 即：距離≤2的位置不能同時有兩支炮兵

### 輸入規格
- 第一行：N 和 M（N≤100，M≤10）
- 接下來 N 行：每行 M 個字符（P 或 H）

### 輸出規格
- 輸出最多能部署的炮兵支數

### 範例
```
輸入：
3 3
PPP
PPP
PPP

輸出：
3
```

## 演算法分析

### 核心思想

使用 **動態規劃 + 位元遮罩** 策略：

1. **為什麼使用位元遮罩？**
   - M ≤ 10，所以每行最多10個位置
   - 每行的部署方案可以用一個10位的二進制數表示
   - 2^10 = 1024 種可能的遮罩，可以枚舉

2. **狀態設計**
   ```
   dp[i][mask1][mask2] = 最大炮兵數
   
   - i：當前處理的行索引
   - mask1：第 i-1 行的部署遮罩
   - mask2：第 i 行的部署遮罩
   ```
   
   需要跟蹤兩行的原因：
   - 檢查相鄰行的衝突（距離=1）
   - 檢查距離為2的行的衝突

3. **約束檢查**

   **同行約束（橫向）：**
   - 同一行中的任意兩個炮兵距離必須 > 2
   - 例如：位置 0 和位置 2，距離=2，違反約束

   **行間約束（縱向）：**
   - 相鄰行間，同一列不能同時有炮兵
   - 距離為2的行間，同一列也不能同時有炮兵

### 時間複雜度

```
總複雜度：O(N × (2^M)^2 × M)
        = O(N × 4^M × M)

由於 M ≤ 10：
- 4^10 = 2^20 ≈ 100萬
- 整體複雜度在可接受範圍內
```

### 空間複雜度

```
O(2^M × 2^M) = O(4^M)
用於儲存DP狀態
```

## 實現細節

### 關鍵函數

#### 1. `count_bits(mask: int) -> int`
計算遮罩中 1 的個數（即該行的炮兵數）
```python
# 使用 Brian Kernighan 算法
count = 0
while mask:
    mask &= mask - 1
    count += 1
```

#### 2. `is_valid_row(mask: int, m: int) -> bool`


#### 3. `can_coexist(mask1: int, mask2: int) -> bool`
檢查兩個相鄰行是否相容
```python
# 相鄰行距離為1，同一列不能同時有炮兵
# 只需檢查：mask1 & mask2 == 0
```

#### 4. `solve_artillery(n: int, m: int, grid: List[str]) -> int`
主求解函數，實現動態規劃

**流程：**
1. 預處理：找出所有合法的行遮罩
2. 初始化：處理第 0 行
3. 轉移：逐行處理第 1 到 n-1 行
4. 返回：找出 DP 表中的最大值

## 常見錯誤

### 1. 理解攻擊範圍
- **正確：** 距離≤2 的位置會互相攻擊
- **錯誤：** 距離<2 的位置才會互相攻擊

### 2. 忽視距離為2的行間約束
- **常見錯誤：** 只檢查相鄰行的衝突，忽視第 i-2 和第 i 行的衝突
- **正確做法：** 跟蹤兩行的狀態，同時檢查

### 3. 遺漏平原限制
- **正確：** 只能在平原（'P'）上部署炮兵
- **常見錯誤：** 遺漏對地形的檢查

## 優化建議

### 1. 預處理兼容性矩陣
```python
# 預先計算哪些遮罩對可以同時出現
compatible[mask1][mask2] = can_coexist(mask1, mask2)
```
